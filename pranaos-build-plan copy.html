Phase 0 Data Model & Infrastructure
Before any screen gets built, the core data model and infrastructure must exist. This is the foundation everything else plugs into.
Depends on: Nothing — this is ground zero

INFRA Infrastructure Tasks
Non-screen work that must exist before any UI can function.
Developer Tasks
  • Database setup: PostgreSQL with row-level security (RLS) policies for multi-tenant isolation. Every query must scope to partner → partition → department
  • Auth system: OAuth2 (Google, GitHub) + email/password. JWT tokens with embedded tenant context (partner_id, partition_id, role)
  • API framework: REST or tRPC. All endpoints require auth middleware that enforces tenant boundaries
  • Encryption layer: Integration credentials and API keys must be encrypted at rest (AES-256). Decrypt only at runtime for agent execution
  • File storage: S3-compatible for logos, brand assets, uploaded docs. Namespaced by /partner_id/partition_id/
  • Ragie integration: SDK setup for knowledge base creation, document indexing, and retrieval. One Ragie collection per KnowledgeStore
  • Stripe integration: Products for plans (Starter, Growth, Enterprise), metered billing for partitions and usage, Checkout Sessions for self-serve signup
  • Email service: Transactional emails (invites, billing) with partner-branded templates. Support custom from-name and logo injection
  • Real-time: WebSocket or SSE for live activity feeds, decision notifications, agent status updates
  • Background jobs: Queue system for agent execution, knowledge indexing, billing calculations, email sending

C-06 Universal Chat Client Admin User P0✓ Mockup
Full-screen "ask anything" chat with conversation sidebar and smart intent routing. Analyzes queries to route to the best agent (Final Sale, Paid Ads, Email Drafter, Revenue Reporter) or knowledge base. Shows routing indicator → typing animation → rich response with source attribution.
Dependencies
  • Agent execution engine
  • Ragie retrieval for knowledge base queries
  • AgentDeployment records to know which agents are available
  • Department-scoped access (user only sees agents/knowledge their department allows)
Features
  • Full-screen layout: Left sidebar (280px) with conversation list + search + "new chat" button. Main area with messages + input bar. Fills viewport height
  • Conversation sidebar: Each item shows title, preview text, routed-agent badge (colored), and timestamp. Active state highlighted. Click to load. Search bar to filter
  • Empty state: Center-aligned "Ask anything about [Client Name]" + 4 suggested prompt chips based on active agents. Clicking a chip sends it as a message
  • Smart routing: Intent detection classifies query → routes to best agent. Visible "Routing to [Agent]…" spinner before typing begins. Supports: inventory → Final Sale, ads → Paid Ads, vendor → Knowledge Base, email → Email Drafter, revenue → Revenue Reporter, fallback → Knowledge Base
  • Message bubbles: User messages right-aligned in client-primary color. AI responses left-aligned with agent avatar + white card. Rich formatting: bold, lists, links, code. Avatar varies by agent
  • Source attribution: Below each AI response: colored badge with agent name + data source text. Links to open agent GUI or source document
  • Typing indicator: Three bouncing dots with agent avatar while response generates
  • Input bar: Auto-resizing textarea (up to 120px), send button, Enter to send / Shift+Enter for newline. Hint: "Routes to the best agent automatically"
Developer Tasks
  • Build chat UI: message list with auto-scroll, auto-resizing textarea, send button, rich markdown rendering
  • Build routing layer: classify incoming query → select best agent → show routing indicator → pass to agent execution → stream response
  • Implement streaming responses (SSE or WebSocket) with typing indicator during generation
  • Build RAG query path: if no specific agent matches, query Ragie knowledge base → return with citations
  • Build source attribution component: agent badge + data source + link to agent GUI or source doc
  • Build conversation persistence: store messages per user per partition, load history, search conversations
  • Build conversation sidebar: list, search, new chat, active state management
  • Build suggested prompts: generate 4 prompts from active agents' capabilities on empty state
  • Scope all queries by department: only route to agents/knowledge the user's department can access
  • Widget mode (P1): floating chat bubble on C-04/C-05 that opens a panel version of this interface
